## Watch-outs / gotchas (project-specific)

1. **`windows.h` macro contamination in `cpp/app/main.cpp`**: enum-qualified access (`Foo::Bar`) breaks after ~7 references with confusing "not declared" errors. Workaround in Slice F: plain integer literals with a pinning test in `test_bench_simulator_wheel_pattern_catalog.cpp`. Don't try to "fix" the literals with an alias — the windows.h preprocessing chokes on it.

2. **`cpp/app/main.cpp` is ~20k LOC single file with a paused refactor** (memory file `project_main_cpp_refactor.md` notes Step 1+2 done, Step 3 paused). Don't add big features inline. Extract to `cpp/app/tabs/` (like `triggers_simulate_panel.{hpp,cpp}`) or `cpp/app/widgets/`.

4. **Qt lifetime rules** (per `CLAUDE.md` "ABI Constraints" memory note): never delete widgets in signal handlers (use `hide()` instead). `QTreeWidget::clear()` also crashes in handlers — use show/hide filtering.

5. **`resources/` is untracked** — large binary firmware trees (Speeduino, Ardu-Stim, Airbear, RusEFI snapshots, decompiled TunerStudio). Do NOT commit. Add to `.gitignore` if you want it tracked-as-ignored.

## Build quick-start

### Vendor single-header deps (one-time, any platform)

```bash
mkdir -p cpp/third_party/doctest cpp/third_party/nlohmann
curl -L -o cpp/third_party/doctest/doctest.h \
  https://raw.githubusercontent.com/doctest/doctest/master/doctest/doctest.h
curl -L -o cpp/third_party/nlohmann/json.hpp \
  https://github.com/nlohmann/json/releases/latest/download/json.hpp
```

### Tests-only build (any platform — fast)

```bash
cmake -S cpp -B build -G "MinGW Makefiles"  # or "Unix Makefiles" / "Ninja"
cmake --build build --target tuner_core_tests
./build/tuner_core_tests        # .exe on Windows
```

### Qt app build (Windows, validated path)

Requires Qt 6.7.3 built from source at `C:/Qt/6.7.3-custom` + MinGW UCRT 15.2.

```bash
cmake -S cpp -B build-app -G "MinGW Makefiles" \
  -DTUNER_BUILD_APP=ON -DTUNER_CORE_BUILD_TESTS=OFF \
  -DCMAKE_PREFIX_PATH=C:/Qt/6.7.3-custom
cmake --build build-app --target tuner_app -j 4
./build-app/tuner_app.exe
```

### Qt app build (Linux, code-complete but unvalidated locally)

```bash
sudo apt install build-essential cmake qt6-base-dev qt6-tools-dev libgl1-mesa-dev
cmake -S cpp -B build-app -G "Unix Makefiles" \
  -DTUNER_BUILD_APP=ON \
  -DCMAKE_PREFIX_PATH=/usr/lib/x86_64-linux-gnu/cmake/Qt6
cmake --build build-app -j$(nproc)
```
